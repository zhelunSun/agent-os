# Templates

Copy-paste templates for all Agent OS file types.

---

## Finding Entry (JSONL line)

```json
{"id":"AF-YYYYMMDD-NNN","date":"YYYY-MM-DD","source":"daily-audit","repo":"<project>","severity":"P1","status":"open","title":"<concise one-liner>","owner":"<agent-or-human>","evidence":["<path-or-ref>"],"next_action":"<what needs to happen>","verification":"<how to confirm resolved>","tags":["<optional>"],"updated_at":"YYYY-MM-DDTHH:MM:SS"}
```

### Example

```json
{"id":"AF-20260601-001","date":"2026-06-01","source":"code-review","repo":"sheaf-ai","severity":"P1","status":"open","title":"Auth module missing test coverage for edge cases","owner":"primary-agent","evidence":["src/auth.py","tests/"],"next_action":"Add edge case tests for token expiry and refresh","verification":"pytest tests/test_auth.py — all pass","tags":["testing","auth"],"updated_at":"2026-06-01T09:00:00"}
```

## Context Pack

```markdown
# Context Pack: <label>

**Date**: YYYY-MM-DD
**Task**: <what this pack is for>
**Generated by**: <agent name>

## Current Goal

<1-2 sentences>

## Must-Read Files

- `<path>` — <why>

## Open Blockers

- <open P0/P1 findings relevant to this task>

## Active Repos / Worktrees

- <repo> — <status>

## Forbidden Actions

- <things NOT to do>

## Expected Output

- <success criteria>
```

### Example

```markdown
# Context Pack: auth-refactor

**Date**: 2026-06-01
**Task**: Refactor auth module to support token refresh
**Generated by**: primary-agent

## Current Goal

Refactor `src/auth.py` to add token refresh logic. Must maintain backward compatibility with existing CLI.

## Must-Read Files

- `src/auth.py` — main module to refactor
- `tests/test_auth.py` — existing tests to preserve
- `.agent-os/findings.jsonl` — open auth-related findings

## Open Blockers

- AF-20260601-001: Missing test coverage for edge cases

## Active Repos / Worktrees

- sheaf-ai — clean, on main branch

## Forbidden Actions

- Do not change the public CLI interface
- Do not introduce new dependencies

## Expected Output

- Refactored `src/auth.py` with token refresh support
- New test cases for edge scenarios
- All existing tests still pass
```

## Flight Record (JSONL line)

```json
{"run_id":"FR-YYYYMMDD-HHMMSS","timestamp":"YYYY-MM-DDTHH:MM:SS","agent":"<name>","automation":"<optional automation name>","task":"<what was asked>","context_summary":"<inputs consumed>","decisions":["<key decision and why>"],"files_changed":["<path>"],"commands_run":["<command>"],"verification":"<how verified>","failures":["<errors>"],"downstream_impact":"<what this affects>","findings_created":["AF-..."],"findings_updated":["AF-..."]}
```

## Experience Entry (JSONL line)

```json
{"id":"EX-YYYYMMDD-NNN","date":"YYYY-MM-DD","source":"<finding-or-incident>","learning":"<what was learned>","classification":"rule|test|gate|prompt|doc","target":"<where to apply>","status":"draft|approved|implemented|verified","evidence":["<refs>"]}
```

### Classification Decision Tree

```
Does it prevent a bug? → test
Does it block a release? → gate
Does it constrain agent behavior? → rule
Does it change automation instructions? → prompt
Otherwise → doc
```
